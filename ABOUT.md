# ABOUT JD AUTOMATION

## Why This Project Exists

JD Automation was not created as a software project.

It started as an operational improvement initiative.

Working in distribution and warehouse operations, I constantly encountered repetitive tasks, duplicate data entry, manual document handling, unnecessary emails, and processes that depended heavily on tribal knowledge.

Instead of accepting these inefficiencies as "the way things have always been done", I began building small tools to simplify daily operations, reduce errors, and save time for the team.

What started as a simple Truck Log importer eventually evolved into a collection of automation tools integrated with SILA and JD Flow.

---

## Vision

My goal is not to replace people.

My goal is to eliminate repetitive work so people can focus on decision making, customer service, problem solving, and operational excellence.

Every minute saved from manual work is a minute that can be invested in activities that create real value.

---

## Core Principles

### Simplicity

Technology should make work easier, not more complicated.

If a solution requires extensive training or creates additional steps, it is not a good solution.

### Efficiency

Every process should be questioned.

If a task is repeated dozens of times every day, it is a candidate for automation.

### Reliability

Automation should reduce errors, increase consistency, and provide traceability.

### Continuous Improvement

There is always a better way.

Every tool in this repository is part of an ongoing effort to improve operations one step at a time.

## Current Projects

### JD Intake Lite

Automates Truck Log import into SILA Pre Entrada.

Features:

* Excel import
* Automatic pallet and box detection
* Reference capture
* Reduced manual data entry
* GitHub-based version management

---

### SILA Tracking + Docs

Provides operational visibility directly inside SILA.

Features:

* Tracking detection
* ODC hover information
* Quick document access
* Operational shortcuts

---

### JD Flow Dispatch

Connects warehouse operations with the dispatch process.

Features:

* Dispatch workflow management
* Document tracking
* Status visibility
* Historical records
* Operational KPIs
